🚀 Deployment Guide

Step-by-step instructions for deploying the Sovereign Mesh

🎯 Prerequisites

Before deploying, ensure you have:

  • ✅ Google Cloud account with billing enabled
  • ✅ Access to Google Cloud Shell
  • ✅ Basic knowledge of Kubernetes (optional but helpful)
  • ✅ GitHub account (for webhook integration)

⚡ Quick Deploy (Recommended)

1. Open Google Cloud Shell

Navigate to console.cloud.google.com and click the Cloud Shell icon in the top right.

2. Run the One-Line Installer

Copy and paste this command:

curl -fsSL https://me10101-01.github.io/deploy.sh | bash
3. Wait for Initialization

The script will create the complete Sovereign Mesh structure in ~/sovereign-mesh/

✅ Success! Your Sovereign Mesh infrastructure is now ready.

🔧 Manual Deployment

Step 1: Create Directory Structure

mkdir -p ~/sovereign-mesh/{queen,valoryield,llm-council,services}
cd ~/sovereign-mesh

Step 2: Deploy Queen Orchestrator

# Navigate to queen directory
cd ~/sovereign-mesh/queen

# The queen.js file should already be created by the setup script
# If not, download it from:
curl -O https://me10101-01.github.io/queen.js

# Install dependencies and start
npm install
npm start
⚠️ Note: Queen will run on port 3000 by default. Use Cloud Shell's Web Preview to access it.

Step 3: Access Queen via Web Preview

  1. Click the Web Preview button in Cloud Shell (top right)
  2. Select Preview on port 3000
  3. You should see: {"service":"Strategickhaos Queen","version":"1.0.0","status":"online"}

Step 4: Deploy to Kubernetes (Optional)

# Create namespace and deploy
kubectl apply -f k8s-deploy.yaml

# Check deployment status
kubectl get pods -n sovereign-mesh

# Get external IP (may take a few minutes)
kubectl get service -n sovereign-mesh

🧠 LLM Council Setup

Docker Compose Deployment

# Navigate to LLM Council
cd ~/sovereign-mesh/llm-council

# Start all models
for llm in llama1 llama2 llama3 llama4 llama5; do
  cd $llm
  docker-compose up -d
  cd ..
done

# Verify all models are running
docker ps
✅ All 5 Llama models should now be running on ports 5001-5005

Test LLM Council

# Run the council orchestrator
python3 llm-council/main.py

# Test individual models
curl http://localhost:5001/health
curl http://localhost:5002/health
curl http://localhost:5003/health

🔗 Integration Setup

GitHub Webhook

  1. Get your Queen's external IP or URL
  2. Go to your GitHub repository settings
  3. Navigate to Webhooks → Add webhook
  4. Set Payload URL to: https://your-queen-url/webhooks/github
  5. Content type: application/json
  6. Select events you want to receive

Zapier Integration (Academic Signals)

  1. Create a new Zap in Zapier
  2. Trigger: Email Parser (for SNHU emails)
  3. Action: Webhooks by Zapier
  4. POST to: https://your-queen-url/signals/academic
  5. Include parsed email data in the payload

Thread Bank Integration

# Configure financial signals endpoint
export THREAD_BANK_WEBHOOK="https://your-queen-url/signals/financial"

# Test the connection
curl -X POST $THREAD_BANK_WEBHOOK \
  -H "Content-Type: application/json" \
  -d '{"event":"test","amount":100}'

🛡️ DNS Configuration

Point Custom Domain to Queen

  1. Get your LoadBalancer IP: 
    kubectl get service queen -n sovereign-mesh
  2. In your DNS provider, create an A record:
    • Name: queen
    • Type: A
    • Value: [LoadBalancer IP]
  3. Wait for DNS propagation (5-30 minutes)
  4. Test: curl https://queen.strategickhaos.ai/health
⚠️ Security: Add SSL/TLS certificate using cert-manager or Google Cloud Load Balancer

🆘 Troubleshooting

Queen won't start

  • Check Node.js is installed: node --version
  • Verify port 3000 is not in use: lsof -i :3000
  • Check logs: npm start

LLM models failing

  • Ensure Docker is running: docker ps
  • Check disk space: df -h
  • Review logs: docker logs [container-id]

Kubernetes deployment issues

  • Check cluster status: kubectl cluster-info
  • View events: kubectl get events -n sovereign-mesh
  • Describe pod: kubectl describe pod [pod-name] -n sovereign-mesh
